## Researched

| Topic | Finding | Link |
|---|---|---|
| Binance access from US IPs | `.com` host returns HTTP 451 (RFC 7725); `.vision` mirror works from same IP; both resolve to `ap-northeast-1` IP space | [ADR-0001](../decisions/0001-binance-vision-mirror.md), [analysis notebook](../../analysis/binance_data_stream_explore.ipynb) |
| Binance `@depth` WebSocket fan-out cadence | Unbatched `@depth` flushes at ~1 Hz at the WS layer with on the order of 130 matching-engine events bundled per frame; `@depth@100ms` gives 10x finer WS-layer cadence | [ADR-0010](../decisions/0010-binance-dual-stream-depth-100ms-plus-trade.md), [collector/exchanges/binance.py](../../collector/exchanges/binance.py) docstring |
| OKX hosting and vantage point | OKX runs on Alibaba Cloud HK (non-AWS-native); `ap-east-1` gives a "near but not co-located" data point, the most interesting non-AWS slot in the cross-venue analysis | [ADR-0009](../decisions/0009-five-region-collector-deployment.md) |
| Latency-measurement methodology | `E - local_receive` deltas usable as a sanity baseline before PTP infra arrives; carries tens to hundreds of ms of NTP skew on a laptop, so shape across messages is informative but absolute value is not | [tests/smoke_binance.py](../../tests/smoke_binance.py), [analysis notebook](../../analysis/binance_data_stream_explore.ipynb) |
