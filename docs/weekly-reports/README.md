# Weekly Reports

Per course rule, the team leader submits weekly reports to Canvas and Box. Drafts live here in version control so the team has a shared editable surface and graders can see the trail in the GitLab repo as well.

## Naming convention

`YYYY-Www.md` where `Www` is the ISO week number. Example: `2026-W17.md` for the week of 2026-04-20 to 2026-04-26.

## Format

Use [`template.md`](template.md) as the starting point.

## Cadence

- Drafted by team leader (currently TBD) and submitted to Canvas + Box every Tuesday
- Committed to this directory at the same time as the Canvas + Box submission
- Reviewed by team async on Discord between submissions

## What goes in a weekly report

- **What was shipped** (commits, ADRs, quick tasks completed)
- **What was researched** (findings, with links to evidence in the repo)
- **What was decided** (link to relevant ADRs)
- **What's next** (next week's plan)
- **Blockers** (waiting on professor, infrastructure, teammates, external)
- **Hours per teammate** (rough estimate; supports individual-attribution course rule)

The CHANGELOG.md and ADR index are the source data; weekly reports are a curated digest.
